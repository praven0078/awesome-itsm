# Contribution Guidelines

Please note that this project is released with a [Contributor Code of Conduct](code-of-conduct.md). By participating in this project you agree to abide by its terms.

## Adding to this list

Before submitting a pull request, please ensure:

- The item is not already on the list
- The item is actively maintained (not archived, deprecated, or missing documentation)
- The item is genuinely useful to ITSM practitioners — not just tangentially related
- You have added the item in the correct category in alphabetical order where possible
- Your entry follows the format: `- [Name](link) - Description.`
- The description starts with a capital letter and ends with a full stop
- The description does not begin with "A", "An", or "The"
- The description does not mention that it is on the list — e.g. no "A curated list of..."
- The link goes to the most useful page, not necessarily the homepage

## Format

```
- [Item Name](https://example.com/) - Short description of the item and why it is useful to ITSM practitioners.
```

## Removing items

Items may be removed if they become unmaintained, deprecated, or no longer genuinely useful. Open an issue if you believe an item should be removed and explain why.

## Updating your pull request

If maintainers ask for changes, please update your pull request. Pull requests that become stale without response will be closed.
